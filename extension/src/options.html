kkongdon YouTube Script Capture 설정

유튜브 영상 자막·메타데이터를 Markdown/CSV/HTML/TXT로 저장합니다. 설정 후 저장을 눌러주세요.

저장 파일명

사용 가능한 토큰: {channel} {date} {title} {video_id}
모든 파일은 다운로드 폴더 안의 YouTube-Capture 폴더에 저장됩니다.

자막 언어 우선순위

체크된 언어를 위→아래 순서로 시도합니다. 사람이 작성한 자막이 자동 생성보다 우선합니다.

AI 공급자 선택

자막 자동 정리·AI 요약 분석을 사용하려면 AI 공급자를 켜세요. 헬퍼와 해당 CLI 설치 필요.

한국어 번역·요약 품질이 중요하면 qwen2.5:7b 강력 권장 (~4.4GB). 3b는 더듬거리는 한국어 자주 발생.
설치: ollama pull qwen2.5:7b · 빠른 영문 처리만: qwen2.5:3b · 더 큰 모델: llama3.1:8b
기본값: claude-haiku-4-5-20251001 (빠름, 토큰 절약). 품질 우선이면 claude-sonnet-4-6 입력. 비워두면 기본 Haiku 사용.

대본 품질 향상

AI 없이 즉시 적용. 비용 없음. 사용자 사전으로 흔한 ASR 오인식 단어 교정 가능.
자주 보는 영상 주제의 용어를 추가하세요. 캡처할 때마다 자동 적용됩니다.
~/.youtube-capture/transcript_hook.py 파일을 만들면 캡처 시 자동 실행됩니다. 예시 스크립트: helper/examples/transcript_hook.py. 외부 라이브러리·로컬 LLM 자유롭게 활용 가능.
자막 전체를 한 번 LLM에 보냄. 토큰 부담이 가장 큰 작업.
엔티티(인물·기술·개념)와 층위(무엇·왜·어떻게·시사점)로 나눠 구조화된 분석 리포트를 생성합니다. 자막 전체를 LLM에 보내므로 AI 공급자 설정 필요 (10~60초 소요).

STT — 자막 없는 영상 자동 변환

Whisper CLI 필요 (pip install -U openai-whisper + ffmpeg). 영상이 정상 재생 중이어야 합니다.
