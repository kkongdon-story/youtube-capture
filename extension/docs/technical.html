확장 ID (고정)

이 확장은 manifest.json의 "key" 필드로 ID를 고정했습니다. 누가 이 코드로 압축해제 로드해도 동일 ID:

kjdgcjakmgocegklcnkanbpjigfajkal

install.mjs는 이 ID로 NM 매니페스트의 allowed_origins를 설정합니다.

헬퍼 수동 호출 (디버깅)

NM 프로토콜 직접 호출 — health 메시지 보내고 응답 확인:

node -e "
const { spawnSync } = require('child_process');
const path = require('path');
const helper = path.join(process.env.USERPROFILE, '.youtube-capture/bin/youtube-capture-helper.cmd');
const msg = JSON.stringify({ id: 't', action: 'health' });
const buf = Buffer.alloc(4 + msg.length);
buf.writeUInt32LE(msg.length, 0);
buf.write(msg, 4, 'utf8');
const res = spawnSync(helper, [], { input: buf, timeout: 8000, shell: true });
const out = res.stdout || Buffer.alloc(0);
if (out.length >= 4) {
  const len = out.readUInt32LE(0);
  console.log(out.slice(4, 4 + len).toString('utf8'));
}"

로그·진단

헬퍼 로그 — 토큰·비밀번호는 절대 기록하지 않음:

Get-Content "$env:USERPROFILE\.youtube-capture\helper.log" -Tail 50

tail -50 ~/.youtube-capture/helper.log

확장 콘솔 로그:

  1. chrome://extensions/ 열기
  2. "유튜브 스크립트 캡쳐" 카드에서 "서비스 워커" 링크 클릭
  3. DevTools Console 열림 — [YTC] 시작 로그 확인

보안 모델

  • NM 호스트의 allowed_origins는 우리 확장 ID 하나만 허용 — 다른 확장/사이트는 헬퍼 호출 불가
  • 헬퍼는 action 화이트리스트 (health, summarize, transcribe, pythonHook)
  • 사용자 텍스트는 stdin으로 전달, spawn 인자 배열로만 명령 구성 — 셸 인젝션 차단
  • Windows에서 .cmd는 shell:true 필요하지만 args 배열은 우리 제어값뿐 (사용자 입력 없음)
  • 로그에 토큰·비밀번호 절대 기록 안 함

실험적 / 알려진 이슈

  • Codex on Windows: stdin/stdout이 TTY가 아닐 때 출력이 빈 경우가 있음 (현재 디버깅 중)
  • 한글 OneDrive 경로: 헬퍼 소스를 한글 경로에 두면 Chrome NM spawn 실패 — install.mjs가 자동으로 ~/.youtube-capture/helper-src/로 복사
  • Ollama 모델 첫 호출: 메모리 로딩 시간 5~30초 발생

제거(Uninstall)

node ~/.youtube-capture/helper-src/install.mjs --uninstall

이후 chrome://extensions/에서 확장 자체를 삭제하면 완전 제거. 사용자 데이터 폴더(~/.youtube-capture/)는 수동으로 삭제.
